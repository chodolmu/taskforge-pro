# Status — 진행 상황 표시

프로젝트의 전체 진행 상황을 한눈에 볼 수 있는 트리로 보여준다.

## 사전 조건

- `_workspace/project-plan.json`
- `_workspace/execution-state.json` (실행 시작 후)

## 출력 형식

```
프로젝트: 크롬 공룡게임 클론
상태: 진행중 | 시작: 2026-04-06 | 경과: 2시간 15분

마일스톤 1: 기본 게임 루프              [████████░░] 75%
  ├─ 스프린트 1.1: 캔버스 세팅          ✅ 완료 (검증 통과)
  │   ├─ HTML 뼈대 만들기              ✅ haiku  (32초, $0.001)
  │   ├─ 게임 루프 구현                ✅ sonnet (2분, $0.04)
  │   └─ 키보드 입력 처리              ✅ sonnet (1분 48초, $0.03)
  ├─ 스프린트 1.2: 캐릭터              🔄 진행중
  │   ├─ 공룡 렌더링                   ✅ sonnet (1분 30초, $0.03)
  │   ├─ 점프 물리                  → 🔄 실행중 (sonnet)
  │   └─ 숙이기 동작                   ⏳ 대기
  └─ 스프린트 1.3: 장애물              ⏳ 대기

마일스톤 2: 점수 시스템                 ⏳ 대기
마일스톤 3: 폴리싱                     ⏳ 대기

진행률: 4/12 태스크 완료 (33%)
비용: $0.101 | haiku: $0.001 / sonnet: $0.10 / opus: $0.00
```

## 상세 모드

`/status detail` 또는 `/status m1-s1-t2`로 특정 항목의 상세 정보:

```
태스크: 게임 루프 구현
상태: ✅ 완료
모델: sonnet | 소요: 2분 15초 | 비용: $0.04

계획:
  requestAnimationFrame 기반 게임 루프 구현.
  deltaTime 계산, update/render 분리.

Handoff:
  gameLoop() 함수 구현 완료. 60fps 타겟.
  update(dt)와 render(ctx) 분리 구조.
  
  설계 결정: deltaTime 기반 물리 (프레임 독립적)
  다음 참고: ctx는 전역 변수로 캐시됨

검증: build ✅ | typecheck ✅
```

## 실패 태스크 표시

실패한 태스크가 있으면 눈에 띄게 표시:

```
  ├─ 충돌 감지                   ❌ 실패 (2회) — 빌드 에러
  │   → `/retry m1-s2-t3` 또는 `/skip m1-s2-t3`
```
