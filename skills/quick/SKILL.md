# Quick — 빠른 실행

discover→plan 과정 없이 바로 작업을 실행한다. 작은 수정, 버그 수정, 단일 파일 작업에 적합.

## 언제 사용하나

- 파일 하나 수정/생성
- 간단한 버그 수정
- 스타일/CSS 변경
- 설정 파일 수정
- 작은 기능 추가

## 언제 사용하지 않나

큰 작업은 `/discover`부터 시작하는 것을 추천:
- 여러 파일에 걸친 변경
- 아키텍처 결정이 필요한 작업
- 여러 단계가 필요한 기능

## 사용법

```
/quick 로그인 버튼 색상 파란색으로 변경
/quick README에 설치 방법 추가
/quick package.json에 lodash 의존성 추가
```

## 실행 흐름

### 1. 작업 분석

요청을 분석하여:
- 무엇을 변경할지 파악
- 관련 파일 확인 (파일 트리 스캔)
- 난이도 판정 (easy/medium/hard)

### 2. 스킬 필요 여부 판단

작업에 도메인 전문지식이 도움되면 `harnesses/`에서 관련 스킬을 자동 로드한다.
예: 보안 관련 수정 → api-security-checklist 스킬 주입

### 3. 즉시 실행

- 깨끗한 컨텍스트에서 실행
- 변경 사항 적용
- 자동 검증 (build, lint 등)

### 4. 결과 보고

```
✅ 완료: "로그인 버튼 색상 변경" (haiku, 12초, $0.001)
   변경: src/components/LoginButton.css
   검증: lint ✅
```

## 선택적 기능

### 토론 (--discuss)

```
/quick --discuss API 응답 형식 변경
```

실행 전에 간단한 논의를 진행한다. 방향이 불확실할 때 유용.

### 리서치 (--research)

```
/quick --research 최적 캐싱 전략 적용
```

실행 전에 코드베이스를 분석하여 기존 패턴을 파악한다.

## 주의사항

- Quick 모드로 실행한 작업은 handoff를 생성하지 않는다
- 프로젝트가 진행 중이면 execution-state에 기록하지 않는다
- 큰 작업을 Quick으로 하려고 하면 "/discover로 시작하는 걸 추천합니다"라고 안내
