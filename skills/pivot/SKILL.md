# Pivot — 방향 전환 (Opus PM)

프로젝트 중간에 방향을 바꿔야 할 때, 남은 계획 전체를 재설계한다.
Opus 모델로 실행한다.

## /plan:edit vs /pivot

| /plan:edit | /pivot |
|-----------|--------|
| 태스크 몇 개 추가/삭제/수정 | 마일스톤 구조부터 재설계 |
| 기존 계획의 틀 유지 | 남은 계획 전면 재구성 |
| "이 태스크 빼줘" | "이 방향으로 바꿔줘" |

## 동작

1. 사용자에게 변경 방향을 듣는다
   - "멀티플레이어 추가하고 싶어"
   - "웹앱 대신 데스크톱 앱으로 바꿔"
   - "이 기능은 필요 없고, 대신 이걸 넣고 싶어"

2. Opus가 분석:
   - 이미 완료된 작업 확인 (되돌리지 않음)
   - 진행중인 스프린트 처리 방법 결정
   - 남은 마일스톤/스프린트/태스크 전면 재설계

3. 새 계획 제시:
   - 완료된 부분은 유지
   - 변경된 부분은 명확히 표시
   - 트리 형태로 before/after 비교

4. 사용자 승인 후 `project-plan.json` 업데이트

## 주의사항

- 이미 완료된 태스크의 결과물은 유지한다. 롤백하지 않는다.
- SpecCard도 갱신이 필요하면 `spec-card.json`도 업데이트한다.
- 승인 후 `/plan:approve` 없이 바로 실행 가능 상태가 된다 (이미 승인된 프로젝트이므로).
