# Cost — 비용 요약

프로젝트 실행에 사용된 비용을 모델별, 단계별로 요약하고, 남은 비용을 예측한다.

## 사전 조건

- `_workspace/execution-state.json`

## 데이터 수집

각 태스크 실행 시 기록되는 데이터:
- 사용 모델 (haiku/sonnet/opus)
- 실행 시간
- 토큰 사용량 (input/output)
- 비용 (execution-state.json의 각 태스크 cost 필드)

PM 호출(plan, refresh, pivot)과 검증(validate, audit) 비용도 별도로 추적한다.

## 예상 남은 비용 계산

남은 태스크의 비용을 모델별 평균 단가로 예측:

```
예상 비용 = Σ (남은 태스크의 모델별 평균 비용)

모델별 평균 비용 계산:
  - 완료된 haiku 태스크의 평균 비용 × 남은 haiku 태스크 수
  - 완료된 sonnet 태스크의 평균 비용 × 남은 sonnet 태스크 수
  - 완료된 opus 태스크의 평균 비용 × 남은 opus 태스크 수

아직 실행된 태스크가 없는 모델은 기본 추정치 사용:
  - haiku: $0.002/태스크
  - sonnet: $0.04/태스크
  - opus: $0.10/태스크
```

## 출력 형식

```
비용 요약 — 크롬 공룡게임 클론

모델별:
  haiku:  3회 실행,  $0.003  (1%)   | 평균 $0.001/태스크
  sonnet: 8회 실행,  $0.32   (78%)  | 평균 $0.04/태스크
  opus:   2회 실행,  $0.08   (20%)  | 평균 $0.04/태스크
  ─────────────────────────
  합계:   13회,      $0.403

단계별:
  마일스톤 1: $0.35 (12/12 태스크 완료)
  마일스톤 2: $0.053 (1/8 태스크 — 진행중)
  마일스톤 3: 미시작

기타:
  PM (계획 수립):  $0.12
  검증:           $0.05
  ─────────────────────────
  총 비용:         $0.573

예상 남은 비용:
  haiku  × 5개 = ~$0.005
  sonnet × 9개 = ~$0.36
  opus   × 2개 = ~$0.08
  검증/PM       = ~$0.05
  ─────────────────────────
  예상 남은:      ~$0.50
  예상 총합:      ~$1.07
```

## 데이터 부족 시

execution-state.json에 cost 필드가 없으면:
- "비용 데이터가 아직 기록되지 않았습니다. 태스크를 실행하면 자동으로 기록됩니다."
- 있는 데이터만으로 가능한 범위의 요약을 표시
