# Execute All — 스프린트 자동 실행 (Wave 병렬)

현재 스프린트의 남은 태스크를 멈추지 않고 연속 실행한다.

## 사전 조건

- `/execute`와 동일

## 동작

### Wave ���반 병렬 실행 (NEW)

태스크에 `wave` 필드가 있으면 wave 순서로 실행한다:

```
Wave 1: [t1, t2, t3] — 의존성 없음, 병렬 실행
  ↓ (모두 완료 후)
Wave 2: [t4, t5] — Wave 1에 의존, 병렬 실행
  ↓ (모두 완료 후)
Wave 3: [t6] — Wave 2에 의존, 단일 실행
```

1. 같은 wave의 태스크들은 Agent 도구로 **병렬 실행**
2. wave 내 모든 태스크 완료 후 다음 wave로 진행
3. wave 필드가 없는 태스크는 의존성 순서대로 **순차 실행**

### 실행 흐름

각 태스크마다 `/execute`와 동일한 흐름 수행:
- 컨텍스트 조립 → 모델 배치 → 실행 → acceptance criteria 확인 → handoff → 검증

### 실패 처리

- 태스크 실패 시에도 멈추지 않고 다음 태스크로 진행
- 단, 실패한 태스크에 의존하는 태스크는 건너뜀 (skipped)
- 같은 wave에서 하나가 실패하면 해당 태스크만 실패 처리, 나머지는 계속 진행

### 결과 요약

스프린트 내 모든 태스크 완료 후:

```
스프린트 1.1 "캔버스 세팅" 완료

  Wave 1 (병렬):
    ✅ HTML 뼈대 만들기 (haiku, 32초)
    ✅ CSS 스타일 설정 (haiku, 28초)
  Wave 2 (병렬):
    ✅ 게임 루프 구현 (sonnet, 2분 15초)
    ✅ 키보드 입력 처리 (sonnet, 1분 48초)
  Wave 3:
    ✅ 통합 연결 (sonnet, 1분 12초)

  소요: 4분 35초 (병렬 효과: 순차 대비 38% 단축)
  비용: $0.15
  검증: build ✅ typecheck ✅
  Acceptance: 전체 통과

  → `/validate`로 스프린트 통합 검증을 실행하세요
  → `/execute:all`로 다음 스프린트를 계속할 수도 있어요
```

## 중단

사용자가 중간에 "멈춰", "중단", "잠깐" 등을 말하면 현재 태스크 완료 후 멈춘다.
이후 `/execute`로 한 개씩 또는 `/execute:all`로 다시 연속 실행할 수 ��다.

## 주의사항

- 하나의 스프린트 단위로만 자동 실행한다. 마일스톤 전체를 한 번에 돌리지 않는다.
  이유: 스프린트 사이에 검증과 계획 갱신이 필요하기 때문.
- 스프린트가 끝나면 반드시 멈추고 검증을 안내한다.
