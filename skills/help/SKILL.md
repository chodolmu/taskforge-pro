# Help — 사용법 안내

TaskForge Pro의 전체 기능과 현재 상황에 맞는 다음 단계를 안내한다.

## 실행 시

### 1. 현재 상태 감지

프로젝트 상태 파일을 확인하여 현재 어느 단계인지 파악:

| 파일 존재 여부 | 상태 | 추천 다음 단계 |
|---------------|------|---------------|
| 아무것도 없음 | 시작 전 | `/discover`로 프로젝트 정의 |
| spec-card.json만 | 정의 완료 | `/plan`으로 계획 수립 |
| project-plan.json (미승인) | 계획 수립 완료 | `/plan:approve`로 승인 |
| execution-state.json | 실행 중 | `/execute`로 다음 태스크 또는 `/status`로 현황 |
| 스프린트 완료 상태 | 검증 필요 | `/validate` 또는 `/verify` |
| 마일스톤 완료 상태 | 감사 필요 | `/audit` |
| 전체 완료 | 마무리 | `/wrap` |

### 2. 상황별 안내 표시

```
## TaskForge Pro — 현재 상태

📍 프로젝트: [이름]
📊 진행: 마일스톤 1/3 — 스프린트 2/4 — 태스크 5/12

### 지금 할 수 있는 것
- `/execute` — 다음 태스크 실행: "게임 루프 구현" [medium/sonnet]
- `/execute:all` — 남은 스프린트 전체 자동 실행
- `/status` — 전체 진행 상황 보기
- `/discuss` — 다음 스프린트 논의

### 전체 명령어

#### 프로젝트 시작
| 명령어 | 설명 |
|--------|------|
| `/discover` | 대화로 프로젝트 정의 |
| `/plan` | 마일스톤/스프린트/태스크 분할 |
| `/plan:edit` | 계획 수정 |
| `/plan:approve` | 계획 승인 |

#### 실행
| 명령어 | 설명 |
|--------|------|
| `/execute` | 다음 태스크 실행 |
| `/execute:all` | 스프린트 자동 실행 |
| `/quick` | 계획 없이 빠른 실행 |
| `/discuss` | 실행 전 논의 |

#### 검증 (3가지 — 역할이 다릅니다)

> `/validate` = AI가 코드를 자동 검증 (빌드, 테스트, 코드 리뷰)
> `/verify` = 사용자가 직접 눈으로 확인 (대화형 UAT)
> `/audit` = 마일스톤 종합 감사 (교차 대조 + 회귀 검증)

| 명령어 | 설명 |
|--------|------|
| `/validate` | goal-backward 검증 |
| `/verify` | 대화형 UAT |
| `/audit` | 마일스톤 회귀 검증 |

#### 모니터링
| 명령어 | 설명 |
|--------|------|
| `/status` | 진행 상황 트리 |
| `/cost` | 비용 요약 |

#### 적응
| 명령어 | 설명 |
|--------|------|
| `/refresh` | 계획 갱신 |
| `/pivot` | 방향 전환 |
| `/retry` | 재시도 |
| `/skip` | 건너뛰기 |

#### 하네스
| 명령어 | 설명 |
|--------|------|
| `/browse-harness` | 100개 하네스 카탈로그 |
| `/use-harness` | 하네스 직접 실행 |

#### 기타
| 명령어 | 설명 |
|--------|------|
| `/resume` | 새 세션에서 이어받기 |
| `/wrap` | 프로젝트 마무리 |
```

### 3. 처음 사용자 안내

프로젝트 파일이 없으면 초보자 가이드를 표시:

```
## TaskForge Pro 시작하기

1️⃣ `/discover` — 만들고 싶은 것을 대화로 정의합니다
2️⃣ `/plan` — AI가 작업을 자동으로 쪼갭니다
3️⃣ `/plan:approve` — 계획을 확인하고 승인합니다
4️⃣ `/execute` — 하나씩 실행합니다 (반복)
5️⃣ `/wrap` — 완료!

작은 작업은 `/quick`으로 바로 시작할 수도 있습니다.
하네스만 쓰고 싶다면 `/browse-harness`로 카탈로그를 살펴보세요.
```
