# Wrap — 프로젝트 마무리

모든 마일스톤이 완료된 후, 최종 QA를 수행하고 프로젝트를 마무리한다.

## 사전 조건

- 모든 마일스톤이 완료 상태
- 완료되지 않은 마일스톤이 있으면 경고하고 진행 여부 확인

## 동작

### 1. 최종 QA (Opus)

Opus가 프로젝트 전체를 점검:

- SpecCard의 모든 기능이 구현되었는지
- 빌드/실행이 정상인지
- 코드 품질 전반
- 알려진 이슈 목록 (handoff들에서 수집)

### 2. 미해결 이슈 정리

모든 handoff의 `knownIssues`를 수집하여 정리:

```
미해결 이슈:
1. [m1-s2-t3] 더블 점프 시 가끔 떨림 발생 — 낮은 우선순위
2. [m2-s1-t2] 하이스코어가 1000점 넘으면 UI 겹침

수정이 필요하시면 `/plan:edit`으로 수정 태스크를 추가할 수 있어요.
```

### 3. 완료 보고서 생성

`_workspace/project-report.md`:

```markdown
# 프로젝트 완료 보고서

## 프로젝트: [이름]
## 완료일: [날짜]

## 요약
- 마일스톤: [n]개 완료
- 태스크: [n]개 완료, [n]개 건너뜀, [n]개 실패 후 수정
- 소요 시간: [시간]
- 총 비용: $[금액]

## 모델 사용량
- Opus: [n]회 ($[금액])
- Sonnet: [n]회 ($[금액])
- Haiku: [n]회 ($[금액])

## 구현된 기능
1. [기능 A] ✅
2. [기능 B] ✅
3. [기능 C] ✅

## 기술 스택
[...]

## 알려진 이슈
[...]

## 파일 구조
[프로젝트 파일 트리]
```

### 4. 마무리 안내

```
프로젝트 "크롬 공룡게임 클론"이 완료되었습니다! 🎉

완료 보고서: _workspace/project-report.md
총 비용: $1.02 | 소요: 4시간 30분

수고하셨습니다!
```
