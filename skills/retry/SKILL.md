# Retry — 태스크 재시도

실패한 태스크를 재시도한다. 이전 실패 원인을 컨텍스트에 포함하여 같은 실수를 반복하지 않게 한다.

## 동작

1. 대상 태스크 확인:
   - `/retry` → 가장 최근 실패한 태스크
   - `/retry m1-s2-t3` → 특정 태스크

2. 실패 원인을 handoff에서 로드

3. 컨텍스트에 실패 원인 추가:
   ```
   이전 시도에서 실패한 원인:
   - [빌드 에러: Cannot find module 'xxx']
   - [이전 시도의 접근 방식과 왜 실패했는지]
   
   이번에는 위 문제를 피해서 작업하라.
   ```

4. `/execute`와 동일한 흐름으로 재실행

5. retry_count 증가. 2회 연속 실패 시:
   ```
   ⚠️ 이 태스크가 2번 연속 실패했습니다.
   
   실패 원인: [요약]
   
   선택지:
   - `/retry` — 한 번 더 시도 (다른 접근으로)
   - `/skip` — 이 태스크를 건너뛰고 다음으로
   - 직접 수정 후 `/handoff` — 사용자가 직접 코드를 수정
   ```
