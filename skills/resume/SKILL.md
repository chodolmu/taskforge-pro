# Resume — 프로젝트 이어받기

새 세션에서 이전 작업을 이어서 진행한다.
전체 히스토리를 읽지 않고, 필요한 맥락만 빠르게 로드한다.

## 동작

1. `_workspace/` 디렉토리에서 프로젝트 상태 로드:
   - `spec-card.json` — 프로젝트 정의
   - `project-plan.json` — 작업 계획
   - `execution-state.json` — 실행 상태

2. 현재 상태 파악:
   - 프로젝트가 어떤 단계에 있는지 (계획중/실행중/검증중)
   - 마지막으로 완료된 태스크
   - 실패/건너뛴 태스크
   - 다음 실행할 태스크

3. 가장 최근 handoff 로드:
   - 마지막 완료 태스크의 handoff
   - 최근 설계 결정사항

4. 요약 출력:

```
프로젝트 이어받기: 크롬 공룡게임 클론

진행률: 8/24 태스크 완료 (33%)
마지막 작업: "점프 물리" (✅ 완료, sonnet)
현재 위치: 마일스톤 1 > 스프린트 1.2 > 태스크 3/3

최근 설계 결정:
- deltaTime 기반 물리 (프레임 독립적)
- Canvas context 전역 캐시

다음 태스크: "숙이기 동작" [medium/sonnet]

→ `/execute`로 다음 태스크를 실행하세요
→ `/status`로 전체 현황을 확인하세요
```

## 주의사항

- `_workspace/`가 없으면: "프로젝트가 없습니다. `/discover`로 새 프로젝트를 시작하세요"
- 계획은 있지만 승인 전이면: "`/plan:approve`로 계획을 승인해주세요"
- 계획 없이 spec-card만 있으면: "`/plan`으로 작업 계획을 세워주세요"
