# Skip — 태스크 건너뛰기

특정 태스크를 건너뛰고 다음으로 넘어간다.

## 동작

### 1. 대상 태스크 확인
- `/skip` → 현재/가장 최근 실패한 태스크
- `/skip m1-s2-t3` → 특정 태스크

### 2. 건너뛰기 사유 확인

사용자에게 사유를 물어본다:

```
이 태스크를 건너뛰는 이유를 선택해주세요:
1. 수동으로 이미 처리함
2. 지금은 불필요함
3. 다른 이유 (직접 입력)
```

사유가 자명한 경우 (실패 후 "/skip" 호출 등)는 물어보지 않고 자동으로 "failed → skipped"로 기록.

### 3. 의존성 영향 분석

이 태스크에 의존하는 후속 태스크가 있는지 확인:

```
⚠️ 이 태스크를 건너뛰면 다음 태스크도 영향받습니다:
- m1-s2-t4 "충돌 감지" (이 태스크에 의존)

선택:
1. 의존 태스크도 함께 건너뛰기
2. 의존 태스크는 유지 (의존성 제거)
3. 취소
```

### 4. 상태 갱신

승인 시:
- 태스크 상태를 `skipped`로 변경
- `skipReason` 필드에 사유 기록
- `skippedAt` 타임스탬프 기록
- execution-state.json 갱신

```json
{
  "id": "m1-s2-t3",
  "status": "skipped",
  "skipReason": "manual",
  "skippedAt": "2026-04-07T..."
}
```

skipReason 값:
- `manual` — 수동으로 이미 처리함
- `not_needed` — 지금은 불필요함
- `failed` — 실패 후 건너뛰기
- `blocked` — 의존 태스크가 건너뛰어져서 함께 건너뜀
- `custom: {사유}` — 사용자 직접 입력

### 5. 완료 안내

```
태스크 "충돌 감지"를 건너뛰었습니다. (사유: 지금은 불필요함)
→ `/execute`로 다음 태스크를 진행하세요
→ 나중에 복원하려면 `/plan:edit`로 상태를 변경할 수 있어요
```
